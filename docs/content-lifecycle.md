# Content Lifecycle Engine V1

Дата: 2026-07-11.

## Цель

Content Lifecycle Engine — единый слой управления состоянием контента АПГ. Он не является отдельным архивом новостей и не привязан к одной коллекции.

Подключённые ресурсы: `news`, `events`, `promotions`, `partners`, `experts`, `prizes`, `customTasks`, `banners`.

## Статусы

Единый список:

1. `draft` — черновик
2. `moderation` — на модерации
3. `scheduled` — запланировано
4. `published` — опубликовано
5. `completed` — завершено
6. `archived` — архив
7. `deleted` — первый этап удаления
8. `trash` — корзина перед окончательным удалением

Архив не удаляет документ и не очищает медиа, комментарии, просмотры, лайки, статистику, связи и историю.

## Совместимость

Движок нормализует существующие поля `lifecycle.status`, `lifecycleStatus`, `contentStatus`, `status`, `archived`, `deleted`, `deletedAt`, `active`. Старые документы продолжают работать без обязательной массовой миграции.

## Запись изменений

При смене статуса backend пишет `lifecycleStatus`, `contentStatus`, `lifecycle`, `lifecycleHistory[]` и audit log. Для новостей дополнительно сохраняется запись в `newsChangeHistory`.

## Автоматические рекомендации

V1 не запускает тяжёлые фоновые сканы на каждом открытии. Рекомендации вычисляются в админке поверх уже загруженных данных.

- опубликованное прошедшее мероприятие → `completed`;
- завершённое мероприятие старше 60 дней → предложить `archived`;
- опубликованная новость старше 180 дней → предложить `archived`.

## Восстановление

Восстановление — обычный переход `archived` → `published`. Документ не пересоздаётся, статистика и медиа остаются на месте.

## Удаление

Удаление разделено: `archived` → `deleted` → `trash` → окончательное удаление. В V1 удаление событий и призов переведено на первый этап `deleted`.

## Поиск

Центр контента поддерживает активные, черновики, модерацию, запланированные, завершённые, архив, удалённые и все материалы. Поиск работает внутри выбранного ресурса и выбранного представления.

## Локи

Локи не смешивает архив с обычными рекомендациями. Архивные и завершённые материалы подключаются к рассуждателю, когда запрос явно похож на поиск истории: архив, прошлый год, старые материалы, когда проходило, прошедшие мероприятия, итоги.

## Backend API

Единый endpoint: `POST /api/admin-actions`.

Actions:

- `lifecycle:overview`
- `lifecycle:transition`
- `lifecycle:bulk-transition`

Новых отдельных endpoint для архива не создаётся.
