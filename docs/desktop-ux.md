# Desktop UX 1.0

Desktop UX 1.0 — этап визуального и UX-доведения существующих Workspace Core, Desktop Workspace, Business Hub и пользовательского режима. Новый функционал не добавлялся: изменения направлены на ощущение законченного desktop/SaaS-продукта.

## Найденные UX-проблемы

1. Пользовательский режим на desktop выглядел как мобильное приложение шириной `480px`, растянутое в центре большого экрана.
2. В Desktop Workspace использовался `GlassPanel` как обычная карточка, хотя этот компонент является экранным контейнером с `minHeight: 100svh`. Из-за этого header, sidebar, status bar и виджеты создавали ощущение больших тёмных слоёв и пустых областей.
3. Dashboard был слишком равномерным: много одинаковых панелей, слабая иерархия, мало информации сразу после открытия.
4. Правая колонка с Локи воспринималась как заглушка, а не как рабочий контекст.
5. Workspace background ощущался как отдельная чёрная плёнка поверх интерфейса, а не как часть рабочей среды.

## Что изменено

### Desktop User Mode

Пользовательский режим остался простым городским приложением, но получил desktop-адаптацию:

- общий контейнер расширен до `1240px` на desktop;
- первый экран использует две колонки: hero + быстрые действия;
- подборки “Что интересного сегодня” на desktop показываются сеткой, а не мобильной горизонтальной лентой;
- новости и мероприятия на desktop располагаются в двухколоночной сетке;
- мобильная версия продолжает использовать прежний компактный layout.

### Desktop Workspace

- `GlassPanel` больше не используется как внутренняя карточка/панель Workspace.
- Header, sidebar, status bar и workspace panels стали обычными glass surfaces без `minHeight: 100svh`.
- Уменьшены ширины служебных колонок, чтобы центральная рабочая область получила больше места.
- Фон Workspace заменён на более мягкий desktop-gradient без ощущения затемняющего overlay.

### Dashboard

Dashboard стал плотнее и информативнее:

- welcome widget показывает сегодняшнюю сводку;
- добавлены виджеты `Статистика` и `Бизнес`;
- сетка виджетов перешла на 12-column layout;
- первый экран лучше заполняет пространство без больших пустых зон.

### Локи

Правая колонка теперь воспринимается как часть рабочего места:

- блок Локи показывает контекстные рекомендации;
- добавлен блок “Сегодня” с быстрыми числовыми сигналами;
- CTA изменён на “Спросить Локи”, без ощущения плавающего ассистента.
