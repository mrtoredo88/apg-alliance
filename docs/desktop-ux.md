# Desktop UX 1.0

Desktop UX 1.0 — этап визуального и UX-доведения существующих Workspace Core, Desktop Workspace, Business Hub и пользовательского режима. Новый функционал не добавлялся: изменения направлены на ощущение законченного desktop/SaaS-продукта.

## Найденные UX-проблемы

1. Пользовательский режим на desktop выглядел как мобильное приложение шириной `480px`, растянутое в центре большого экрана.
2. В Desktop Workspace использовался `GlassPanel` как обычная карточка, хотя этот компонент является экранным контейнером с `minHeight: 100svh`. Из-за этого header, sidebar, status bar и виджеты создавали ощущение больших тёмных слоёв и пустых областей.
3. Dashboard был слишком равномерным: много одинаковых панелей, слабая иерархия, мало информации сразу после открытия.
4. Правая колонка с Локи воспринималась как заглушка, а не как рабочий контекст.
5. Workspace background ощущался как отдельная чёрная плёнка поверх интерфейса, а не как часть рабочей среды.

## Что изменено

### UX Convergence

После сравнения User Mode и Workspace общий вывод такой:

- User Mode ощущается живым за счёт hero-блока, мягкой визуальной глубины, сценарного повествования и понятного “что делать дальше”.
- Workspace сильнее архитектурно: роли, Business Hub, панели, shortcuts, контекст, рабочие списки.
- Для convergence в Workspace перенесён принцип User Mode: не начинать экран с равных карточек, а строить его как рабочий день с главным центром внимания.

Dashboard теперь строится по сценарию:

1. рабочий hero `Сегодня в АПГ`;
2. что требует внимания;
3. главное сегодня;
4. мой бизнес;
5. статистика;
6. новости;
7. мероприятия;
8. последние действия;
9. быстрые действия и Локи.

Центром Dashboard стал рабочий hero-блок: он показывает уведомления, мероприятия, новости, рекомендации Локи и активный бизнес-профиль. Это сближает визуальный язык Workspace с User Mode, но сохраняет SaaS-логику работы.

### Desktop UX 1.1

Поверх первого прохода добавлена доводка профессионального поведения:

- левая навигация сгруппирована по рабочим зонам: `Работа`, `Контент`, `Каталоги`, `Система`;
- добавлены hover/focus-состояния и краткие подсказки для collapsed sidebar;
- header упрощён: shortcut hint стал компактным, role/user-mode/notifications занимают меньше места;
- добавлен shortcut overlay по `?`;
- добавлены быстрые клавиши `⌘2 / Ctrl 2` для Business Hub и `⌘3 / Ctrl 3` для Content;
- добавлено desktop context menu по правому клику на свободной области Workspace;
- Dashboard получил виджеты `Сегодня` и `Требует внимания`;
- восстановлен общий `APG2_PROFILE.goldGradient`, чтобы active/gold-состояния выглядели одинаково в Workspace и User Mode.

### Desktop User Mode

Пользовательский режим остался простым городским приложением, но получил desktop-адаптацию:

- общий контейнер расширен до `1240px` на desktop;
- первый экран использует две колонки: hero + быстрые действия;
- подборки “Что интересного сегодня” на desktop показываются сеткой, а не мобильной горизонтальной лентой;
- новости и мероприятия на desktop располагаются в двухколоночной сетке;
- мобильная версия продолжает использовать прежний компактный layout.

### Desktop Workspace

- `GlassPanel` больше не используется как внутренняя карточка/панель Workspace.
- Header, sidebar, status bar и workspace panels стали обычными glass surfaces без `minHeight: 100svh`.
- Уменьшены ширины служебных колонок, чтобы центральная рабочая область получила больше места.
- Фон Workspace заменён на более мягкий desktop-gradient без ощущения затемняющего overlay.

### Dashboard

Dashboard стал плотнее и информативнее:

- welcome widget показывает сегодняшнюю сводку;
- добавлены виджеты `Сегодня`, `Требует внимания`, `Статистика` и `Бизнес`;
- сетка виджетов перешла на 12-column layout;
- первый экран лучше заполняет пространство без больших пустых зон.

### Локи

Правая колонка теперь воспринимается как часть рабочего места:

- блок Локи показывает контекстные рекомендации;
- добавлен блок “Сегодня” с быстрыми числовыми сигналами;
- CTA изменён на “Спросить Локи”, без ощущения плавающего ассистента.

### Loki Operating Workspace

Следующий этап переводит Локи из роли виджета в роль центра управления Workspace:

- первый экран открывается через briefing Локи: приветствие, реальные сигналы дня, заполненность профиля и рекомендации;
- hero Dashboard больше не является декоративной карточкой — это начало рабочего разговора;
- правая колонка переименована в `AI Workspace` и содержит Локи, Today, внимание, рекомендации, историю, чат и быстрые действия;
- `⌘L`, пункт меню “Локи” и кнопки “Спросить Локи” больше не открывают отдельный экран, а добавляют ответ в AI Workspace;
- центральные overlay/окна для Локи в Desktop Workspace не используются;
- User Mode не изменён: там Локи остаётся обычным помощником городского приложения.
