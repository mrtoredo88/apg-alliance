# Loki Personality Engine V1

## Принцип

Personality Engine применяется только после формирования полезного ответа или успешного действия. Контекстное наблюдение никогда не заменяет результат и не выводится в критической ситуации.

```text
Полезный результат
      │
Situation classifier
      │
Safety gate ── blocked → только деловой ответ
      │
Mode + role + screen + time + memory
      │
Frequency gate
      │
Anti-repeat selector
      │
Короткое наблюдение после ответа
```

## Режимы

- `professional` — только деловой ответ, personality phrase всегда отключена.
- `friendly` — редкие мягкие комментарии.
- `charismatic` — более выразительные, но короткие наблюдения.

Режим переключается пользователем внутри пространства Локи и сохраняется вместе с существующими `lokiSettings` локально и через действующий backend sync настроек.

## Контекст

Выбор учитывает:

- время суток;
- роль пользователя;
- текущий экран;
- тип завершённого события;
- историю использованных фраз;
- количество разговоров;
- длительность текущей сессии;
- серию одинаковых действий;
- давность предыдущего визита.

Для 24 продуктовых и административных контекстов зарегистрированы собственные subject/result/place descriptors. Phrase packs отделены от ядра, поэтому новые события, шаблоны и контексты добавляются без изменения selection pipeline.

## События

Поддерживаются успешное действие, deploy, исправление бага, ожидание, генерация изображения, публикация новости, создание мероприятия, долгая и ночная работа, регистрация, первый запуск, достижение, награда, удачный поиск, сложная операция, повторный визит, долгое отсутствие, обнаружение ошибки и открытие админки.

## Safety gate

Фраза блокируется до выбора библиотеки для:

- ошибок авторизации;
- потери данных;
- удаления;
- финансовых операций;
- критических ошибок;
- проблем безопасности;
- контекстов с `critical`, `dataLoss`, `financialOperation`, `destructiveAction` или `securityIssue`.

Дополнительно проверяется текст результата, чтобы критическая ситуация не получила юмор из-за неверной классификации события.

## Анти-повторы

Последние 24 id хранятся в существующей локальной памяти Локи. Выбор исключает последние 12 использованных вариантов. Когда доступные варианты исчерпаны, пул безопасно открывается заново.

Библиотека содержит 1 679 контекстных вариантов. Это произведение отдельных авторских шаблонов и контекстных словарей, а не монолитный список внутри engine.

## Проверка

```bash
npm run test:loki-personality
npm run test:loki
npm run test:admin-assistant
npm run build
```

Тестовая матрица охватывает девять ролей, 17 ключевых экранов, все запрещённые события, критические тексты, профессиональный режим, длинную сессию и отсутствие повторов в доступном пуле.
