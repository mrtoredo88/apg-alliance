# 07 ADMIN PANEL

## Обзор

**URL:** `/#/admin`, `/#/admin-app`  
**Файл:** `src/AdminPanel.jsx` (~5000+ строк, единый компонент с V4.4-оболочкой)  
**Авторизация:** вход из профиля виден для `admin/owner` и legacy VK id; вход в админку проверяется через Firebase session/email+password и `/api/admin-security` с server-side role guard
**UI:** 100% inline styles, темизация через объект `A` (локальные токены для dark theme)

## V4.4 редакционная оболочка

Админка получила единый адаптивный интерфейс:

- на телефоне навигация становится горизонтальным верхним островом
- на ноутбуке и desktop остаётся боковая панель
- Dashboard показывает рабочие KPI: модерация, комментарии, ошибки, пользователей, партнёров, экспертов, события, новости, ключи и QR

## Вкладка «ИИ-импорт»

- Блок «Публичные формы» создаёт токен-ссылки `/submit/:type/:token` для партнёров, экспертов, событий, новостей и призов
- Для каждой ссылки доступны копирование шаблона, копирование URL, готовое сообщение для Telegram/WhatsApp и QR-код
- История `publicFormLinks` показывает тип, дату, открытия, статус (`link_created`, `submitted`, `disabled`) и позволяет повторно скопировать или отключить ссылку
- Публичная форма не требует авторизации пользователя; основная анкета короткая и сразу отправляется, а юридический блок свёрнут по умолчанию и открывается только добровольно или при платном сценарии
- Основная анкета собирает данные для публикации и связи, включая обязательный ИНН для будущей проверки организации и маркировки рекламы
- Карточка сотрудничества хранится отдельно от публичного черновика: `legalProfile`, `legalDocuments`, `legalCheck`, `counterparty`, `cooperationPlan`, `cooperationStatus`, CRM-заготовка под договоры, счета, акты, ЭДО и комментарии
- Реквизиты и документы показываются в админке только ролям `owner`, `super_admin`, `admin`; backend `/api/admin-actions` вырезает закрытые поля для остальных ролей и запрещает им изменять юридические данные
- Статусы карточки сотрудничества: `legal_not_required`, `legal_recommended`, `legal_partial`, `contract_ready`, в интерфейсе отображаются как “юридические данные не требуются”, “рекомендуется заполнить”, “заполнены частично”, “готов к договору”
- Поиск в админке умеет находить заявки по ИНН, названию, директору, телефону и email, если роль имеет доступ к юридической карточке
- После создания/сохранения партнёра с email запускается мастер подключения: `/api/admin-actions` actions `partner:onboarding-check`, `partner:bind-owner`, `partner:send-invite`, `partner:remind-later` проверяют пользователя по `emailIndex/users`, защищают от повторной привязки, пишут `connectionStatus`, `partnerOnboarding`, `partnerInvite`, `partnerConnectionEvents` и выдают доступ к партнёрскому кабинету
- Pipeline запуска партнёра продолжается actions `partner:publish-catalog` и `partner:mark-verified`: backend возвращает мастер публикации на 5 шагов, подробные `readiness.checks`, launch-рекомендации Локи, проверяет readiness минимум 80% и обязательное `publicationConsentAccepted`, публикует `catalogPublished:true`, `active:true`, `searchable:true`, `mapVisible:true`, `lokiVisible:true`, ставит `newPartnerUntil` на 14 дней и идемпотентно создаёт черновик приветственной новости. Push после публикации остаётся предложением администратору, а не автоматической рассылкой. Статус “Проверенный партнёр АПГ” выдаётся отдельным backend-действием после публикации, привязки владельца и проверки контактов.
- Очередь заявок партнёров, экспертов, событий, новостей и призов
- Админ выбирает тип заявки, вставляет текст или загружает TXT/CSV/JSON/MD-файл
- PDF/DOCX/XLSX/изображения можно прикрепить как метаданные заявки; распознавание содержимого подключается следующим шагом через OCR/parser
- Локальный слой Локи раскладывает подписанные поля по структуре, находит телефон/email/ссылки, предлагает категорию, confidence и список недостающих полей
- Заявки сохраняются в `aiImportRequests` через `/api/admin-actions`, без прямой записи из клиента в Firestore
- Кнопка «Создать черновик» создаёт выключенный draft в партнёрах/экспертах/событиях/новостях/призах; публикация остаётся ручным действием редактора

## Вкладка «Доступ»

- Backend: `/api/admin-security`
- Вход требует явный email/password через `/api/admin-login`; старая Firebase-сессия не пропускает в админку автоматически
- `/api/admin-login` проверяет scrypt-хеш пароля и выдаёт Firebase custom token, поэтому админка не зависит от включённого Firebase Password provider
- Проверяет текущую Firebase-сессию администратора и роль через server-side guard
- Показывает текущую роль, список администраторов, матрицу прав и журнал безопасности
- Роли: owner, super_admin, admin, editor, moderator, analyst, partner, expert, user
- Действия для `owner/super_admin`: создание администратора, смена роли, блокировка/разблокировка, отзыв сессий, смена временного пароля
- Новые администраторы создаются в Firebase Auth; пароль не хранится в Firestore и должен быть изменён при первом входе
- `owner` нельзя заблокировать, понизить или удалить через интерфейс
- Все действия пишутся в `adminSecurityLog` и `adminActivity`
- Touch ID / Face ID отображаются как WebAuthn/Passkeys-ready слой; до полноценной регистрации Passkey быстрый вход не обходит email/password

## Вкладка «Аналитика Локи»

- Читает `lokiAnalytics` через `/api/admin-actions`
- Показывает количество диалогов, долю вопросов без ответа, успешность, среднее время ответа
- Отдельные списки: популярные вопросы, непонятые вопросы, частые intents/actions и переходы после рекомендаций
- Непонятый вопрос можно сразу перенести в форму «База знаний Локи» как заготовку ответа
- Запись аналитики идёт best-effort через `/api/user-actions` action `loki:analytics`, поэтому сбой аналитики не ломает диалог

## Вкладка «Ошибки»

### Функционал
- Читает `errorLogs` коллекцию
- Показывает уникальные ошибки с `occurrences`, уровнем, первым/последним появлением, компонентом, маршрутом, версией, пользователем, устройством, браузером и ОС
- Раскрывает stack trace, `stackHash`, связанные действия и историю последних возникновений
- Поиск и фильтры по уровню, версии, маршруту, пользователю, компоненту и статусу

## Moderation: partnership applications

Раздел «Модерация» также читает `aiImportRequests`, чтобы показать заявки из пользовательского сценария подключения партнёров и экспертов из профиля.

Фильтр:

- `Новые заявки на подключение`

Условия попадания:

- `source === 'partnership-flow'`
- или `moderationStatus === 'new_partnership_application'`
- или `crm.lifecycleStage === 'new_partnership_application'`

Опубликованные и отклонённые заявки не показываются в фильтре. Полная обработка карточки остаётся в разделе «ИИ-импорт», куда ведёт кнопка из фильтра.

## Безопасность AdminPanel

**Нет серверной авторизации.** Любой Firebase-авторизованный пользователь технически может открыть `/admin` и использовать форму. Защита:
1. URL `/admin` не опубликован публично (security by obscurity)
2. Firestore Rules разрешают запись только auth пользователям
3. Планируется: проверка `isAdmin` поля в `users/{uid}` документе
