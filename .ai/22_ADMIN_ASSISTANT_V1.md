# Admin Assistant V1

## Назначение

Admin Assistant V1 — постоянный контекстный помощник внутри `AdminPanel`. Это не отдельный экран и не свободный чат. Он объясняет текущий раздел, анализирует уже загруженные данные и переводит администратора к следующему безопасному действию.

## Архитектура

```text
AdminPanel state
  ├── activeTab / filters / search / selected entity
  ├── authenticated role / permissions
  └── existing page data
              │
       AdminContextEngine
              │ immutable context snapshot
     AdminAssistantEngine
       ├── section knowledge
       ├── deterministic insights
       ├── command matching
       └── next-step navigation
              │
     AdminAssistantPanel
       ├── compact summary
       ├── collapse / expand
       ├── pin / auto-collapse
       ├── resize
       └── docked desktop / floating compact layout
```

## Источники данных

V1 не делает самостоятельных Firestore/API запросов. Он получает из `AdminPanel` уже загруженные массивы пользователей, партнёров, экспертов, мероприятий, новостей, комментариев, ошибок, призов, уведомлений, журнала действий и аналитики.

Контекст обновляется через `useMemo` только при изменении вкладки, фильтра, поиска, выбранной сущности, роли, времени загрузки или исходных данных.

## Поддержанные разделы

Знания зарегистрированы для 24 разделов: главная, пользователи, партнёры, эксперты, новости, мероприятия, центр событий, комментарии, модерация, активность и ключи, призы, QR/ротация, аналитика, рассылки, ошибки, доступ и журнал действий, система, реклама, задания, автоматизация, рефералы, база знаний Локи, аналитика Локи и ИИ-импорт.

## Проверяемые факты V1

- Новости: черновики, очередь модерации, отсутствие обложек, давность публикаций.
- Партнёры: отсутствие логотипа, описания, соцсетей/сайта и связанных мероприятий.
- Эксперты: незаполненные профили, отсутствие фото, мероприятий и публикаций.
- Пользователи: незавершённая регистрация, неподтверждённая почта, отсутствие активности 30 дней.
- Мероприятия: отсутствие обложки, организатора, описания и пересечения времени.
- Ошибки: активные, авторизационные, повторяющиеся и критические ошибки.
- Комментарии и модерация: новые элементы, ожидающие проверки.
- Главная: компактная суточная сводка по пользователям, партнёрам, мероприятиям, ошибкам и новостям.

Если нужного поля или набора данных нет, помощник не формирует положительное утверждение и сообщает, что вывод нельзя подтвердить.

## Команды

Поддержаны команды поиска проблем по партнёрам, событиям, новостям, пользователям, комментариям и ошибкам. Результат содержит число найденных объектов и переход в существующий раздел. V1 не изменяет данные и не обходит backend permission matrix. На desktop панель занимает отдельную колонку и не перекрывает рабочую область; на компактном экране используется ограниченный плавающий блок.

## Производительность

- Нет polling и фоновых запросов.
- Расчёты мемоизированы на уровне React.
- Максимальная сложность пересечений событий — `O(n²)` только при изменении массива событий; для текущего административного объёма допустимо.
- Настройки UI сохраняются локально и не содержат персональных данных.

## Проверка

```bash
npm run test:admin-assistant
npm run test:loki
npm run build
```

Автотест проверяет вычисления фактов, честный fallback, команды, навигационные next steps, 24 раздела и роли `owner`, `super_admin`, `admin`, `editor`, `moderator`, `analyst`.
