# Анкета эксперта V2

## Поток

`PublicSubmitPage` подключает отдельный `ExpertQuestionnaire` только для ссылок типа `expert`. Остальные публичные анкеты продолжают работать через существующую универсальную форму.

Анкета разделена на раскрывающиеся блоки: основная информация, услуги, специальное предложение, медиа, тариф, контакты, юридические данные, фотографии и административный комментарий.

## Обязательные поля

Минимальный набор: фамилия, имя, хотя бы одно направление, короткое описание, контактное лицо, телефон и email. Город, ИНН, стоимость услуг, новости и мероприятия не обязательны.

## Тарифы

- `start` и `standard`: основные профильные и контактные поля.
- `premium` и `ambassador`: дополнительно новости, мероприятия и ИНН.

Поля Premium не только скрыты в UI, но и отбрасываются при публикации draft для неподходящего тарифа.

## Расширяемые структуры

- `serviceCatalog: []` — будущий отдельный каталог услуг и цен.
- `futureLegalProfile` — будущие legal type, ОГРН, БИК и расчётный счёт.
- `futureCityProfile` — будущая мультигородная привязка.
- `categories` и `secondaryCategories` — множественные направления.
- `workFormats` — онлайн, офлайн, индивидуально, группы, консультации, выезд и дистанционно.
- `videos` — структурированные ссылки YouTube, VK Видео, Rutube и MAX.
- `otherSocials` — дополнительные социальные ссылки.

## Autosave

Черновик полей и уже загруженных изображений сохраняется локально с ключом одноразового token через debounce 450 мс. Персональные данные не отправляются на backend до явной отправки. После успешной отправки draft удаляется.

## Медиа

Изображения разделены по ролям `avatar`, `logo`, `gallery`, `cover`. Для каждой роли показаны рекомендации по количеству и размерам. Видео валидируется по поддерживаемой платформе и получает предпросмотр, если платформа поддерживается общим `parseVideoUrl`.

## Backend и публикация

Vercel и Fastify используют одинаковый analyzer. Новые массивы и поля нормализуются в draft без выдумывания отсутствующих значений. `aiImportDraftPatch` переносит данные в существующую схему эксперта, а предпросмотр продолжает использовать единый `EntityPreviewCard`. Стоимость услуг в публичную карточку не переносится.

## Проверки

```bash
npm run test:expert-questionnaire
npm run build
```

Автотест проверяет parity двух backend-реализаций, необязательность города и ИНН, массивы категорий и форматов, тарифный доступ, MAX video, будущий каталог услуг, заполненность и данные для карточки.
