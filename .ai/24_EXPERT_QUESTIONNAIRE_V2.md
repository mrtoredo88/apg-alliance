# Тарифные анкеты ИИ-импорта V1

## Поток

`PublicSubmitPage` подключает тарифные анкеты для ссылок типа `partner` и `expert`. Остальные публичные анкеты продолжают работать через существующую универсальную форму.

Сначала выбирается тариф, затем форма показывает только доступные для тарифа поля.

Для бизнеса используются тарифы:
- `start` — Старт;
- `alliance` — Альянс;
- `premium` — Премиум.

Для экспертов используются тарифы:
- `practice` — Практика;
- `ambassador` — Амбассадор.

## Обязательные поля

Эксперт: фамилия, имя, хотя бы одно направление, короткое описание, контактное лицо, телефон и email.

Партнёр: название, категория, короткое описание, подробное описание, контактное лицо, телефон и email.

Город и стоимость услуг не входят в обязательный набор. Необязательная стоимость сохраняется и отображается в профиле, если эксперт её заполнил; позднее она переедет в отдельный каталог услуг. ИНН доступен только тарифам, где есть юридические возможности.

## Тарифы

- Эксперт `practice`: профиль, услуги, контакты, фото, видео и запись.
- Эксперт `ambassador`: дополнительно новости, мероприятия и ИНН.
- Партнёр `start`: базовая карточка, контакты, акция, логотип и фото.
- Партнёр `alliance`: дополнительно онлайн-запись, галерея и видео.
- Партнёр `premium`: дополнительно новости, мероприятия и ИНН.

Поля недоступных тарифов не только скрыты в UI, но и отбрасываются backend analyzer при публикации draft.

## Расширяемые структуры

- `serviceCatalog: []` — будущий отдельный каталог услуг и цен.
- `futureServiceCatalog` — будущие услуги, онлайн-запись, оплата и пакеты.
- `futureScheduleProfile` — будущий календарь и расписание.
- `futureLegalProfile` — будущие legal type, ОГРН, БИК и расчётный счёт.
- `futureCityProfile` — будущая мультигородная привязка.
- `categories` и `secondaryCategories` — множественные направления.
- `audienceTags` — аудитории эксперта для будущих рекомендаций Локи.
- `workFormats` — онлайн, офлайн, индивидуально, группы, консультации и выезд.
- `videos` — структурированные ссылки YouTube, VK Видео, Rutube и MAX.
- `otherSocials` — дополнительные социальные ссылки.

## Единый справочник экспертов

`server-shared/expert-directory.js` — единственный источник категорий для анкеты, backend analyzer, админки, каталога, фильтров, preview и поиска Локи. Он также нормализует legacy aliases (`kids` → `children`), телефон в международный E.164-подобный формат, contact aliases и полный опубликованный профиль. Неизвестная категория блокирует публикацию ИИ-черновика и требует выбрать значение из справочника.

## Autosave

Черновик полей и уже загруженных изображений для партнёра и эксперта сохраняется локально с ключом одноразового token через debounce 450 мс. Персональные данные не отправляются на backend до явной отправки. После успешной отправки draft удаляется.

## Медиа

Изображения разделены по ролям `avatar`, `logo`, `gallery`, `cover`. Для каждой роли показаны рекомендации по количеству и размерам. Видео валидируется по поддерживаемой платформе и получает предпросмотр, если платформа поддерживается общим `parseVideoUrl`.

## Backend и публикация

Vercel и Fastify используют одинаковый analyzer. Новые массивы и поля нормализуются в draft без выдумывания отсутствующих значений. `aiImportDraftPatch` переносит данные в существующие схемы партнёра и эксперта, а предпросмотр продолжает использовать единый `EntityPreviewCard`. Акция, услуги, опыт, необязательная стоимость, телефон, WhatsApp, адрес и часы работы сохраняются до опубликованной карточки.

## Проверки

```bash
npm run test:expert-questionnaire
npm run build
```

Автотест проверяет parity двух backend-реализаций, необязательность города и ИНН, массивы категорий/форматов/аудиторий, тарифный доступ, MAX video, будущие структуры, заполненность и данные для карточек.
